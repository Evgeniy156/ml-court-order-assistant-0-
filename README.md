# ml-court-order-assistant-0-
Система «ИИ-помощник по судебному приказу» автоматизирует полный цикл работы по взысканию бесспорной задолженности за ЖКУ от момента получения данных из бухгалтерии до передачи исполнительного документа в ФССП.

Объектная модель ML-сервиса (ДЗ №1)
Проект содержит объектную модель, разработанную в рамках задания «Спроектировать объектную модель сервиса» на курсе по ML-инженерии.

Модель описывает:

пользователей системы: User, Manager, Admin;

систему балансов и финансовых операций;

работу с ML-моделями и задачами через абстракцию MLModel и MLTask;

доменную область «Судебный приказ»: DebtCase, Calculation, PretrialNotice, CourtOrder, Enforcement.

Принципы ООП

Инкапсуляция
Поля _password_hash и _balance_credits закрыты от прямого изменения.
Доступ к ним выполняется только через методы top_up(), spend(), check_password(), balance.

Наследование
Классы Admin и Manager являются расширениями User.
Общие атрибуты и методы получены из базового класса.

Полиморфизм
Методы can_view_all_cases() и can_approve_topups() определяют права доступа в зависимости от роли.
Код, работающий с пользователями, использует один и тот же интерфейс, а поведение подставляется за счёт разных реализаций.

Назначение

Эта модель служит основой для следующих этапов курса:

проектирование БД через ORM,

реализация REST-интерфейса и Telegram-бота,

интеграция с RabbitMQ,
